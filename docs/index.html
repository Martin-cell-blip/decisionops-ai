A bounded agent loop

The model is deliberately not the calculator or policy engine. This makes the product less magical and more useful: every number is reproducible, every cited fact is inspectable, and every money/price/inventory action has a human gate.

01PlanSelect completeness, matching, quality and opportunity checks.
02Execute toolsCompute facts with deterministic Python policies.
03Apply policyMap facts to a bounded candidate action.
04Narrate + verifyAccept model prose only when [E#] citations are valid.
05Human gateApprove before money, price or inventory moves.
